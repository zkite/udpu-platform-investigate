# Идеальный prompt для Codex (UI)

Сгенерируй изменения для этого проекта, ориентируясь на следующие требования.

## Цели

1. Сделать красивый и простой дизайн интерфейса.
2. Полностью реализовать функционал для раздела/сущности roles.
3. Для остальных разделов/сущностей пока использовать заглушки (минимальная рабочая болванка, без реальной логики).
4. После обновления страницы логин не должен сбрасываться.

## Ожидаемое поведение

- UI выглядит современно, минималистично и аккуратно: понятная типографика, аккуратные отступы, визуальная иерархия.
- Roles: экран списка, создание/редактирование/удаление, валидации, состояние загрузки и ошибок — всё полностью рабочее.
- Остальные разделы: заглушки с понятным текстом, что раздел пока в разработке.
- Сессия пользователя сохраняется между перезагрузками страницы.

## Технические указания

- Не добавляй лишних библиотек.
- Сохраняй текущий стиль проекта.
- Минимизируй изменения вне UI и логики roles.
- Для сохранения логина используй устойчивое хранилище на стороне клиента и корректную инициализацию состояния при старте приложения.

## Критерии готовности

- Роли полностью функциональны.
- Остальные разделы не ломаются и отображают заглушки.
- Обновление страницы не выкидывает пользователя из системы.
- Код легко читается и соответствует текущим паттернам проекта.
